## Задача 1
### ООП. Базовый класс
Разработать класс с одним строковым полем. Создать конструктор копирования. Разработать метод, приписывающий к полю в начало три знака восклицания. Перегрузить метод ToString() для формирования строки из полей класса. 
### Алгоритм решения
Создание базового класса StringWrapper:
1. Поле stringValue (защищённое, тип string) - хранит строковое значение
2. Конструктор с параметром StringWrapper(string value) - инициализирует поле, обрабатывает null
3. Конструктор копирования StringWrapper(StringWrapper anyString) - создаёт копию объекта
4. Метод Exclamation() - добавляет три восклицательных знака в начало строки
5. Переопределённый метод ToString() - возвращает строковое представление объекта
Реализация конструктора копирования:
1. Принимает объект того же типа
2. Копирует значение поля stringValue
Реализация метода Exclamation() - Присваивает полю stringValue значение "!!!" + stringValue

## Задача 2
### ООП. Дочерний класс
Реализовать дочерний класс (его содержание предложить самостоятельно и описать в решении: какой содержательный смысл имеют поля; реализовать конструкторы; предложить и реализовать 2-3 метода). Протестировать все конструкторы и другие методы базового и дочернего классов.
### Алгоритм решения
Создание дочернего класса RegisterString, наследующего StringWrapper
Добавление полей:
1. private string originalValue - для хранения исходной строки
2. private bool isUpper - для хранения текущего режима
Реализация конструкторов:
1. RegisterString(string originalValue, bool isUpper) - вызывает конструктор базового класса через base(originalValue)
2. RegisterString(RegisterString anyString) - конструктор копирования
Реализация методов:
1. ToUpper() - устанавливает isUpper = true и вызывает UpdateStringValue()
2. ToLower() - устанавливает isUpper = false и вызывает UpdateStringValue()
3. UpdateStringValue() - обновляет поле stringValue базового класса
Переопределение метода ToString() - выводит исходное значение, новое значение и текущий режим

# Задание 2
## Задача 1
### Перегрузка операций
Разработать класс Money. Реализовать добавление произвольного количества копеек (uint) к объекту типа Money. Результат должен быть типа Money.
### Алгоритм решения
1. Вычисляем общую сумму в копейках: long total = (long)rubles * 100 + kopeks + kop
2. Вычисляем новые рубли: uint newRub = (uint)(total / 100)
3. Вычисляем новые копейки: byte newKop = (byte)(total % 100)
4. Возвращаем новый объект new Money(newRub, newKop)
При добавлении копеек сумма может превысить 100. Автоматическое преобразование 100 копеек в 1 рубль через деление и остаток

# Задание 3
# Задача 1
### Перегрузка операций
Унарные операции:
++ добавление копейки к объекту типа Money
-- вычитание копейки из объекта типа Money
Операции приведения типа:
uint (явная) – результатом является количество рублей (копейки отбрасываются);
double(неявная) – результатом является копейки, переведенные в рубли, целые рубли отбрасываются, результат <1.
Бинарные операции:
+ Money m, беззнаковоецелое число (лево- и правосторонние операции). Результат должен быть типа Money.
– Money m, беззнаковоецелое число (лево- и правосторонние операции). Результат должен быть типа Money.
### Алоритм решения
1. public static Money operator ++(Money m) - добавляет 1 копейку через метод Add(1)
2. public static Money operator --(Money m) - вычитает 1 копейку через метод Subtract(1)

Реализация операций приведения:
1. public static explicit operator uint(Money m) - возвращает поле rubles
2. public static implicit operator double(Money m) - возвращает kopeks / 100.0

Реализация бинарных операций:
+: public static Money operator +(Money m, uint rub) - увеличивает рубли
-: public static Money operator -(Money m, uint rub) - уменьшает рубли 

Реализация вспомогательных методов:
Add(int kop) - добавляет копейки
Subtract(int kop) - вычитает копейки
Normalize() - приводит копейки к нормальному виду (100 копеек = 1 рубль)
ToString() - форматированный вывод
